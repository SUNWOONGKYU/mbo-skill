# /mbo — 목표에 의한 관리 (Management by Objectives)

> **PO의 철학:**
> "과정은 너의 마음대로 진행해도 좋다. 그러나, 나랑 합의를 해서 사전에 설정한 목표는 무조건 달성해야 한다. 단, 품질은 최고로, 비용은 합리적 수준으로, 투입시간은 적절하게."

이 스킬의 핵심은 **자율과 책임**이다:
- **자율**: 목표 달성 방법은 AI가 자유롭게 선택한다. 과정에 간섭하지 않는다.
- **책임**: 합의된 목표는 반드시 달성한다. 변명이 아니라 결과로 보고한다.
- **품질**: 대충 끝내는 것이 아니라 최고 수준의 결과물을 만든다.
- **효율**: 불필요한 비용(토큰, 시간)을 낭비하지 않는다. 합리적으로 진행한다.

> 어떤 작업이든 실행 전에 `/mbo`를 실행하면:
> 1. 현재 상황을 분석하고
> 2. 목표를 구조화하고
> 3. PO 승인을 받고 (= 합의)
> 4. 목표 기반으로 자율 실행하고
> 5. 완료 시 목표 달성 여부를 보고한다 (= 책임)

---

## 사용법

```
/mbo                    — 대화 맥락에서 자동으로 목표 도출
/mbo 디자인 혁신         — 특정 주제 지정
/mbo report             — 진행 중인 MBO의 달성 여부 보고
```

## PHASE 1: 목표 정의

### Step 1: 맥락 분석

대화 내용에서 다음을 파악:
- **무엇을**: 어떤 작업/변경/개선을 하려는지
- **왜**: 동기/배경/문제점
- **범위**: 어디까지 할 것인지

### Step 2: AS-IS 조사

실제 코드/파일/시스템 상태를 확인:
- 관련 파일 Read
- 현재 동작 상태 확인 (배포 사이트, 빌드 결과 등)
- 문제점/부족한 점 정리

### Step 3: MBO 목표서 작성

아래 양식으로 PO에게 제시:

```markdown

  MBO 목표서 — {작업명}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Step 4: PO 승인 대기

- PO가 승인하면 PHASE 2로 진행
- PO가 수정 요청하면 목표서 수정 후 재제시
- **승인 없이 작업 시작 절대 금지**

## PHASE 2: 실행

### MBO 파일 저장

목표서를 파일로 저장하여 추적:

```
저장 위치: zz_KingFolder/_TalkTodoPlan/YYYY_MM_DD__HH.MM_MBO_{작업명}.md
```

내용: 승인된 MBO 목표서 전문 + 승인 일시

### 작업 실행

- Task 목록 순서대로 실행
- SAL Grid 프로젝트면 SAL Grid 규칙(6단계 루프) 적용
- 일반 작업이면 작업 → 검증 → 기록 순서

### 중간 점검 (선택)

대규모 작업 시 중간에 KPI 현황 보고:

```
MBO 중간 점검 — {작업명}
━━━━━━━━━━━━━━━━━━━
| 지표 | 목표 | 현재 | 달성률 |
|------|------|------|--------|
```

## 핵심 원칙

1. **합의된 목표는 무조건 달성** — 목표 달성이 최우선. 과정은 자유, 결과는 필수.
2. **품질은 최고로** — 대충 끝내는 것은 달성이 아니다. 상품성 있는 결과물이어야 한다.
3. **비용은 합리적으로** — 불필요한 서브에이전트 남발, 중복 작업, 과잉 설계를 하지 않는다.
4. **투입시간은 적절하게** — 빠르되 품질을 희생하지 않는다. 느리되 낭비하지 않는다.
5. **측정 가능한 KPI** — "좋게 만든다"가 아니라 "Lighthouse 80점 이상"처럼 수치화
6. **승인 = 합의** — PO가 목표를 이해하고 동의한 뒤에만 작업. 승인 없이 실행 금지.
7. **과정은 자율** — 목표 달성 방법은 AI가 판단. PO는 과정에 간섭하지 않는다.
8. **결과는 목표 대비** — "뭘 했다"가 아니라 "목표를 달성했다/못했다"로 보고
9. **미달성은 사유와 후속** — 실패도 기록해야 다음에 반복 안 함

## SAL Grid 연동

SAL Grid Dev Suite 프로젝트에서 `/mbo`를 실행하면:
- Stage 시작 전 MBO = 섹션 6.0 프로세스와 동일
- Stage Gate 리포트 = `/mbo report` 결과를 섹션 8.2에 포함
- MBO 파일이 Stage Gate 검증의 목표 달성 여부 근거 자료가 됨
